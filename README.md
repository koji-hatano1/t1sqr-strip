# t1sqr-strip

`t1sqr-strip` is a T1w-based brain masking tool using FreeSurfer's `mri_synthstrip` with squared-intensity adaptive thresholding.

This tool is designed as a T1w-based counterpart to `t2log-strip`.

---

## Version

`t1sqr-strip v2.11`

---

## Overview

While `mri_synthstrip` is a powerful and flexible brain extraction tool, additional mask refinement may be useful for some HCP-style T1w datasets.

`t1sqr-strip` applies squared-intensity transformation for adaptive threshold estimation and generates a refined T1w-based brain mask.

The squared image is used only to estimate thresholds. The final thresholding is applied to the original SynthStrip-derived T1w brain image.

---

## Optimization Workflow

Parameters should be adjusted based on the histogram provided in each subject log.

### 1. Initial `border_num` Selection

In the `# --- Configuration ---` section:

- `border_num=1`: tighter extraction
- `border_num=2`: more conservative extraction

### 2. Fine-tuning via `SD_FACTOR_T1`

Use the histogram in the subject log to adjust `SD_FACTOR_T1`:

- `1.960`: standard starting point
- `2.241`: intermediate
- `2.576`: conservative

> Tip: prioritize avoiding over-stripping.

---

## Usage

### 1. Setup

Edit the configuration in `t1sqr-strip.sh`:

    # --- Configuration ---
    Subjlist="001 002 003"
    BASE_PATH="/path/to/your/project"

    border_num=2
    SD_FACTOR_T1=1.960

### 2. Execution

    chmod +x t1sqr-strip.sh
    ./t1sqr-strip.sh

### 3. Review and Adjust

After execution:

1. Check the histogram in each subject log.
2. Evaluate mask quality visually.
3. Adjust `SD_FACTOR_T1` or `border_num` if needed.
4. Re-run after recovery if necessary.

If the brain is over-stripped, increase `SD_FACTOR_T1` or set `border_num=2`.

If non-brain tissue remains, decrease `SD_FACTOR_T1` or set `border_num=1`.

---

## Recovery

Before re-running with new parameters, restore original files:

    chmod +x recover_t1ss.sh
    ./recover_t1ss.sh

The recovery script restores original files from `_bet.nii.gz` backups and removes temporary files generated by `t1sqr-strip`.

---

## HCP Integration

Designed for HCP-style preprocessing workflows.

The script:

- updates T1w and T2w brain-extracted images
- synchronizes the mask to MNINonLinear space
- creates backups before modifying existing files

---

## QA & Reporting

A summary CSV is generated:

    hss_t1sqr_summary_*.csv

The CSV includes threshold values, SD factors, voxel counts, voxel drop rates, and final mask size.

This file is useful for cohort-level quality assurance.

---

## Viewer

Use the Freeview helper script:

    chmod +x fview_t1ss.sh
    ./fview_t1ss.sh 001

The viewer compares the original backup and current `t1sqr-strip` outputs.

## Prerequisites

Ensure the following are available in your `$PATH`:

- FSL
- FreeSurfer with `mri_synthstrip`
- `bc`
- Freeview
